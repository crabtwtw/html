所有指令在「訊號 Bot」對話框輸入。大部分操作可從訊號訊息底下的按鈕完成,不用記指令。

🚀 開始使用

/start
註冊或重新申請。新用戶第一個指令。
/help
看可用指令說明。

🚪 註冊流程

1
在訊號 Bot 打 /start
2
輸入 BingX UID(Bot 會自動驗證 API)
3
狀態變「待審核」,等管理員審核
4
收到「帳號已開通」通知後即可使用
5
訊號來時會收到訊息 + 下單按鈕

⚙️ 個人設定

/risk <百分比>
每筆訊號的風險比例(餘額的 X%)。沒設用全局預設。
/risk 2 → 每筆風險 2%
/margin <模式>
保證金模式:isolated(逐倉) / cross(全倉)。
/margin isolated
/slotcount <數字>
分倉模式的倉位數。20 = 分 20 倉,每倉 = 餘額 ÷ 20。
/slotcount 20
/autosl <模式>
自動移動止損:tp1(TP1 保本) / trailing(逐步鎖利) / off(關閉)。
/autosl tp1

📋 訊號過濾

/symbol
設定幣種白名單(只接收特定幣的訊號)。
/timeframe 或 /time
設定時間週期過濾(只接 1h / 4h / 全部)。

💰 查詢

/pnl
查自己的損益(今日 / 本月 / 累計)。
/orders
查自己目前的持倉訂單。
/order <symbol>
查單筆訂單詳細(含強平距離)。
/order BTC-USDT

💱 下單操作流程

訊號到達後,Bot 會推訊息給你。以下是按鈕操作流程:

1
收到訊號訊息(含交易對 / 進場 / 止盈 / 止損)
2
選環境:💰 真實(用 USDT)或 🌐 模擬(用 VST)
3
選進場方式:⚡ 立即市價(馬上成交)或 🎯 理想 D 點(預掛單等價)
4
選保證金模式:🗂 分倉(餘額 ÷ N)或 💰 固定金額
5
選出場方式:💯 只下 TP1 100%(保守)或 TP1+TP2 70%/30%(分批)
6
選保本:🔐 TP1 保本(避免回吐)、🔄 移動保本、或關閉讓波段跑
7
按 真實下單 或 模擬下單 送出
所有設定一旦選定就會記住,下次訊號來預設用同樣設定,你可隨時改。

🤖 全自動指令(/auto)

給「全自動會員」用,跑獨立 API 自動接收訊號下單。需先登錄白名單。

📊 狀態

/auto status
看目前模式、逆勢處理、今日失敗筆數。

🔧 模式切換

/auto vst
🔵 只下 VST(模擬) 養數據用,不花真錢。
/auto live
🟢 只下 LIVE(真實) 真金白銀下單。
/auto both
🟣 VST+LIVE 都下 兩個都跑。
/auto off
❌ 全部關閉 訊號來也不下。

🔁 逆勢訊號處理

/auto skip
⏭ 跳過不下(預設)— 逆勢訊號就不下單。
/auto rev
🔁 翻轉下反向 — 訊號做多就改下做空。
/auto raw
➡️ 下原方向 — 不管逆勢,照訊號方向下。

📈 數據 / 診斷

/auto summary
立即推昨日總結(每日 00:00:10 自動推一次)。
/auto verify
對帳檢查 — 用 BingX income API 抓真實 PnL,跟 DB 推算對比。
/auto debug
列昨日訂單明細(診斷用)。
/auto blocklist
看幣黑名單 — 列出失敗過的幣(BingX 不支援、下架等)。
黑名單在訊號失敗時自動加入(只對「下架/不存在」這種錯誤,餘額不足等暫時問題不會擋)。
成功下單會自動移除。看到清單後可至 TV 把那些幣從訊號清單移除,根治洗版問題。

💡 注意事項

狀況說明
訊號來但沒看到可能你的 timeframe / symbol 過濾擋掉了,或被管理員暫停。
下單失敗「保證金不足」BingX 餘額不夠,或設的 risk / leverage 太高。
下單失敗「目前市價已超過 TP」訊號太晚進來,行情已經跑超過止盈點。BingX 會直接拒絕,正常的。
下單失敗「此幣模擬倉暫時無法交易」BingX 該幣 VST 下架,系統自動加黑名單。打 /auto blocklist 看清單。
API 失效找管理員「🔑 重設 API」,然後重新輸入。
真實下單是真錢操作,BingX 餘額會實際進出。新用戶建議先用🌐 模擬練習,熟悉後再用💰 真實。
