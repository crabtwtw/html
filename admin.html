📊 用戶管理

/users 或 /list
列出所有用戶,標題會顯示全站總人數與各狀態分布。可加篩選條件。
/users active(已開通) /users pending(待審) /users rejected(已拒) /users search 王(搜尋)

/user <chat_id>
查單一用戶詳細(API 狀態、設定、損益、操作按鈕)。
/user 12345678
/uid <BingX UID>
用 BingX UID 反查 Telegram 用戶。
/uid 34988913

📢 廣播 / 系統

/bc <內容>
廣播公告給所有 active 用戶。送出前有確認按鈕避免誤發。
/bc 系統將於今晚 23:00 維護 30 分鐘
/maint
系統維護模式開關(暫停訊號廣播 + 公告用戶)。
/gd on/off
訊號群顯示控制。關了訊號群不顯示訊號,但全自動還是會跑。
/regflow on/off
註冊流程開關(是否需要審核才能用)。
/ch
訊號頻道路由切換(A 通道 / B 通道)。
/maint 一打就會通知所有用戶。確定要維護再用,不要拿來測試。

⚙️ 全局預設

/ds 或 /defaultsettings
開啟全局預設面板(下單環境、進場方式、TP 模式、保證金模式等)。
/pf 或 /pushdefaults
把目前全局預設值一鍵推送給所有現有用戶。
/trendfilter
趨勢過濾器(逆勢警告、檢查週期 1h / 4h)。
/trendfilter 1h(寬鬆) /trendfilter 4h(平衡) /trendfilter both(嚴格) /trendfilter off /trendfilter on

📂 訂單管理

/orders [chat_id]
列訂單。沒帶參數查自己,帶 chat_id 查指定用戶。
/orders 或 /orders 12345678
/order <symbol>
查單筆訂單詳細(含強平距離)。
/order BTC-USDT
/resettle
手動觸發結算所有 pending 訂單(平常 3 分鐘自動跑一次)。
/clearpending [天數]
清空待結算訂單,可指定只清超過 N 天的。
/clearpending 或 /clearpending 7
/clearuser <chat_id> [symbol]
清指定用戶所有 pending,可只清某幣。
/clearuser 12345678 BTC-USDT
/fixpending [confirm]
修復 pending 訂單缺漏欄位。沒帶 confirm 是預覽,加 confirm 才執行。
/fixpending(預覽) /fixpending confirm(執行)
/debugorders
查最近 5 筆訂單的 DB 原始資料(除錯用)。

🤖 全自動 /auto

管理員自己也是 auto member 1。完整 /auto 指令請看用戶手冊的全自動章節。

/auto status / vst / live / both / off
查狀態 / 切換下單環境(只下 VST / 只下 LIVE / 都下 / 全關)。
/auto rev / skip / raw
逆勢處理:翻轉 / 跳過 / 下原方向。
/auto summary / verify / debug / blocklist
數據總結 / 對帳 / 訂單明細 / 黑名單。

👤 用戶操作按鈕

在 /user <chat_id> 顯示的用戶卡片上會看到以下按鈕:

按鈕動作
🔑 刪除 API(重設)清掉用戶 API Key,讓他重新輸入。
⏸️ 暫停停止該用戶接收訊號與下單。
▶️ 恢復從暫停恢復。
🗑️ 移除完全刪除用戶,要重新 /start 才能再申請。
✅ 通過 / ❌ 拒絕待審核用戶的審核按鈕。

🚪 用戶審核流程

  1. 用戶在訊號 Bot 打 /start
  2. 用戶輸入 BingX UID,Bot 自動驗證 API Key
  3. 管理員收到「新用戶申請」通知 + 審核按鈕
  4. 點 ✅ 通過 或 ❌ 拒絕
  5. 用戶收到結果通知;通過的可開始接訊號

💡 常見排查

狀況處理
用戶說「訊號收不到」查 /gd 是否開啟;查 /user <id> 是否被暫停;查用戶 timeframe / symbol 過濾。
用戶 API 失效用「🔑 刪除 API」讓他重新輸入。
大量 pending 訂單卡住/resettle 強制結算;或 /clearpending 7 清舊的。
每日總結數字不對新版用 BingX income API 為準。打 /auto verify 對帳檢查。
同個幣一直推失敗訊息該幣會自動進黑名單。/auto blocklist 看清單,然後去 TV 把該幣從訊號移除。
需要全系統暫停/maint 開維護模式,所有用戶會收到暫停通知。
想知道今天哪個用戶賠最多新指令 /losers,直接列今日虧損 Top 5。
想看當天整體成績新指令 /today,當日所有用戶下單統計。
